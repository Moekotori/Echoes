# ECHO Next 26.5.29 / 26.5.30 更新日志

时间范围：2026-05-28 至 2026-05-30  
取材范围：最近提交记录与当前工作区候选改动。当前仍有未提交内容，本文是候选更新说明，最终发布前以实际验收结果为准。

## 一句话版本

这一轮更新重点是：继续保护播放稳定性，同时补强歌词、专辑信息、流媒体收藏、远程音乐库、插件体系和设置页体验；后台任务、诊断和 UI 文案也做了更细的收敛。

## 播放稳定性与诊断

- 启动诊断和播放性能日志更完整，遇到卡顿、启动慢或数据库异常时，更容易从控制台和诊断信息里定位原因。
- 播放性能记录补充更多关键节点，方便后续判断问题来自音频链路、数据库、渲染层还是后台任务。
- 低风险调整多处后台行为，继续避免扫描、远程同步、封面预热、诊断轮询等工作抢占播放热路径。
- 修复桌面歌词锁定状态下的鼠标穿透问题，减少桌面歌词影响正常桌面操作的概率。

## 歌词与专辑信息

- 歌词匹配链路增强：本地歌词读取、查询构造、评分与来源质量记忆继续完善，目标是更少误配、更稳定命中。
- 歌词设置里的显示选项改为可折叠面板，并记住展开状态，常用设置更清爽。
- 专辑详情补充外部评分与更多在线信息展示，MusicBrainz / Wikipedia 等来源的信息可见性更好。
- 专辑详情页增加更多菜单能力，可把整张专辑加入播放队列，也可以直接打开本地专辑所在文件夹。

## 流媒体与收藏

- 新增流媒体收藏导入 / 导出能力，便于迁移或备份跨平台收藏数据。
- YouTube、Bilibili、SoundCloud 等流媒体搜索和收藏链路继续补强，MV 面板、播放栏状态和搜索页体验同步优化。
- 流媒体播放与本地播放的状态提示更明确，减少用户误判“当前到底由谁在播放”。

## 远程音乐库

- 远程来源增加“断开”和“删除”的清晰区分：断开会保留连接配置并清理本地索引；删除会移除本地索引和连接配置，但不会删除服务器音乐文件。
- 远程来源列表、浏览器和后台任务面板继续整理，来源管理、任务暂停、封面预热和专辑合并策略更容易理解。
- 远程封面加载策略与远程专辑合并策略补齐多语言文案，减少界面里中英文混杂。
- 远程后台任务仍按“播放优先”原则推进，同步、封面和元数据补齐不应影响本机播放流畅度。

## 设置页与界面

- 设置页继续重构：通用、播放、联动、插件、外观、关于、危险操作等区域的文案和状态展示更完整。
- 高级播放设置默认收起，并记住用户展开状态，避免普通用户一进设置页就被高风险选项淹没。
- 联动页的登录 / 凭据区默认收起，Spotify、TIDAL、Discogs、在线艺人源、Last.fm 等配置更集中。
- 音频专业信息面板改为可折叠结构，并保留刷新入口，降低视觉噪音。
- 主题计划、侧边栏显示 / 排序、壁纸、更新检查、数据备份、数据库保护等设置项补齐说明和状态。
- 首页、歌曲、歌单、新歌收件箱、历史、连接等页面继续补齐 i18n 文案和空状态。

## 插件与扩展

- 插件系统继续扩展 manifest、权限、默认状态、IPC 和服务层能力。
- 插件页面和设置入口补强，插件运行时、权限边界、播放安全策略和文档入口更清楚。
- 插件 SDK 文档继续更新，方便后续第三方扩展按稳定边界接入。

## 曲库与性能

- 分页标题查询避免在 JS 层做全量排序，降低大曲库场景下的额外开销。
- 艺人合并、元数据读取、扫描队列和曲库核心逻辑继续补充测试，重点保护大库、分页和后台扫描稳定性。
- 数据库健康检查与保护流程补强，坏库、隔离、快照、恢复和诊断导出的状态更明确。
- KGM 解密支持已合入，后续仍需结合真实文件覆盖更多兼容性验证。

## 文档与发布

- README 补充爱发电、插件教程和排障入口，普通用户和插件开发者更容易找到对应说明。
- Linux 构建、插件 SDK、用户指南等文档继续更新。
- Windows 打包如果失败，需要先区分是编译问题还是 `dist\win-unpacked` / `app.asar` 被占用导致的清理失败。

## 已知边界

- 当前工作区仍有未提交改动，尤其是设置页、远程来源、i18n 和多处 UI polish，发布前还需要按实际准备纳入的范围验收。
- Connect / AirPlay / DLNA / HQPlayer 等外部播放方向仍需要真实设备验证，不能只靠模拟测试判断稳定。
- 流媒体搜索与收藏功能依赖外部平台行为，平台接口、地区和账号状态变化都可能影响结果。
- 高级设置、数据库保护、远程来源删除等高影响功能需要继续保持明确确认和可恢复边界。
